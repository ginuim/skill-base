为什么需要 Skill Base？

三个痛点，一个解决方案

痛点 01

公共平台不适合存放私有业务

公司的编码规范、内部 API 文档、业务逻辑不能上传到公共平台，但又需要让 AI 理解这些内容。

私有化部署，数据完全掌控
痛点 02

告别四处复制粘贴

每次换项目、带新人，都要把长长的"系统提示词"和"业务背景"重新发一遍，版本混乱。

像 npm 一样安装和更新团队知识
痛点 03

拒绝重型架构

不想为了管理几个提示词文件就部署复杂的基础设施，K8s、Docker Compose 太重了。

一行 npx 命令即可启动

核心特性

为开发者精心打造的每一个细节

零配置启动

一行命令启动服务，SQLite 自动初始化，开箱即用

npm 般的体验

search / install / publish，熟悉的工作流，零学习成本

快速开始

从安装到使用，只需几分钟

# 一行命令启动服务（推荐指定数据目录） npx skill-base -d ./skill-data -p 8000 # 访问 Web 管理后台 打开浏览器访问 http://localhost:8000 # 首次访问将引导设置管理员账号 # 通过 Web 界面即可浏览、搜索、下载和发布 Skill
# 全局安装 CLI 工具 npm install -g skill-base-cli # 配置你的 Skill Base 服务器地址 skb init -s http://your-server # 登录 skb login
# 搜索可用的 Skills skb search react NAME DESCRIPTION react-best-practice React 开发最佳实践 react-testing React 测试规范 # 安装 Skill 到当前项目 skb install react-best-practice ✓ Installed react-best-practice@v20240320.143022
# 创建一个新的 Skill 目录 mkdir my-team-skill && cd my-team-skill # 创建 SKILL.md 文件（必需） cat > SKILL.md << 'EOF' # My Team Coding Standards 这是我们团队的编码规范... ## 命名规范 - 使用 camelCase 命名变量 - 使用 PascalCase 命名组件 EOF # 发布到 Skill Base skb publish ✓ Published my-team-skill@v20240320.150000
# 安装 Skill 到 Cursor skb install my-skill --ide cursor ✓ Installed to .cursor/skills/my-skill/ # 安装到 Qoder skb install my-skill --ide qoder ✓ Installed to .qoder/skills/my-skill/ # 支持的 IDE: cursor, qoder, windsurf, copilot, claude-code, opencode

设计哲学

Stay Out of the Way — 极简与无侵入

纯粹的搬运工

Skill Base 只做一件事：把 Skill 从服务器搬到你的项目里。不做解析，不做转换，原封不动。

极简优先

能用一个文件搞定的，绝不引入两个。SQLite 单文件部署，就是这么任性。

本地优先

所有数据都在你本地。不想用云？内网也能跑。不想升级？永不更新也能用。
