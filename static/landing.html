为什么需要 Skill Base？

三个痛点，一个解决方案

痛点 01

公共平台不适合存放私有业务

公司的编码规范、内部 API 文档、业务逻辑不能上传到公共平台，但又需要让 AI 理解这些内容。

私有化部署，数据完全掌控
痛点 02

告别四处复制粘贴

每个项目都要手动复制 .cursorrules 文件，版本混乱，更新困难，团队成员各用各的。

像 npm 一样安装和更新
痛点 03

拒绝重型架构

不想为了管理几个提示词文件就部署复杂的基础设施，K8s、Docker Compose 太重了。

一行 npx 命令即可启动

核心特性

为开发者精心打造的每一个细节

⚡

零配置启动

一行命令启动服务，SQLite 自动初始化，开箱即用

🔄

npm 般的体验

search / install / publish，熟悉的工作流，零学习成本

🔒

私有化安全

完全部署在你的内网，敏感规范不外泄

📦

版本控制

每次发布自动生成版本号，支持版本回溯和对比

🌐

双端支持

Web 界面浏览管理，CLI 集成到开发工作流

快速开始

从安装到使用，只需几分钟

# 全局安装 CLI 工具 npm install -g skill-base-cli # 登录到你的 Skill Base 服务器 skb login http://your-server:8000 # 查看帮助 skb --help
# 搜索可用的 Skills skb search react NAME DESCRIPTION react-best-practice React 开发最佳实践 react-testing React 测试规范 # 安装 Skill 到当前项目 skb install react-best-practice ✓ Installed react-best-practice@v20240320.143022
# 创建一个新的 Skill 目录 mkdir my-team-skill && cd my-team-skill # 创建 SKILL.md 文件（必需） cat > SKILL.md << 'EOF' # My Team Coding Standards 这是我们团队的编码规范... ## 命名规范 - 使用 camelCase 命名变量 - 使用 PascalCase 命名组件 EOF # 发布到 Skill Base skb publish ✓ Published my-team-skill@v20240320.150000
# 安装 Skill 到 Cursor skb install my-skill --ide cursor ✓ Installed to .cursor/skills/my-skill/ # 安装到 OpenClaw skb install my-skill --ide openclaw ✓ Installed to .qoder/skills/my-skill/ # 支持的 IDE: cursor, openclaw, cline, windsurf

设计哲学

Build to Delete — 为删除而构建

纯粹的搬运工

Skill Base 只做一件事：把 Skill 从服务器搬到你的项目里。不做解析，不做转换，原封不动。

极轻量、模块化

核心代码不到 2000 行，SQLite 单文件数据库，零外部依赖服务。可以跑在任何能运行 Node.js 的地方。

零技术债

当不再需要时，删掉整个 data 目录就干净了。不污染你的系统，不留下任何垃圾。

最好的工具，是你感觉不到它存在的工具。

— Skill Base 设计理念
